Autonomous Race cars 1/10 the scale, but its 10 times the fun

         

Week 2

16 Nov, 2015 Posted by ARC

    The corridor around the Xlab(Moore 205) was mapped using a hector slam package. Only LiDAR scans were used for mapping purpose and hence we had to add extra features to the corridors.

     Once the mapping was done, the next task was to localize the car in the environment using the saved map. The best approach to localization using laser scans is using a particle filter to match the live laser scan data to the saved map. It needs an approximate position to localize that car that is generally provided using wheel odometry and a defined start position in the map. However, in our case, the car wasn't equipped with wheel encoders and hence we had to find another source of odometry to localize it in the provided map.

     In order to over come this limitation, we used the actual mapping package and allowed it to generate a parallel map in the background that would be discarded and the transforms generated with respect to the initial position were utilized as noisy odometry data. The odometry data was fed to the AMCL ROS package that handled the particle filter, aligning the scans with the map. The final localization output was a combination of the result obtained from noisy odometry and particle filter from AMCL.

     As shown in the video, both the arrows represent the position of robot in different frames. The green arrow is the odometry output from mapping package and the red arrow is the processed odometry output that the car actually utilizes for navigation.

     We were able to navigate properly in the corridors with this localization and a path planning algorithm that we developed. However, the update rate of LiDAR being only 10Hz, the data processing took a lot ofcomputation time reducing the final update rate of position and thus putting an upper cap on the speed limit.
